In a professional design environment, creating high-fidelity prototypes is essential for communicating your ideas to clients, stakeholders, and developers. By adding interactive elements to your designs, you help bring them closer to a real-world implementation. This lab will guide you through creating an interactive, high-fidelity Figma design using the tools and techniques you’ve learned so far. These skills will be invaluable as you work on more complex projects where high-quality, interactive designs are essential.

Learning Objectives

By the end of this lab, you will be able to:

Create a high-fidelity Figma design.

Use Figma’s prototyping features to add basic interactivity.

Submission Requirements

Submit your completed lab using the Start Assignment button on the assignment page in Canvas. Your submission should include:

A link to your Figma project.

Instructions

Step 1: Research and Inspiration

Before starting your design, take some time to explore existing designs in the Figma Community or other design resources for inspiration. You may choose to base your work on an existing design, but make sure to customize it so that it reflects your own creative direction by the time it’s complete.

Tip: Keep the design simple and focus on no more than three individual pages. Consider designing for mobile, which can help reduce the workload and provides a good starting point for beginners.

Step 2: Wireframing

With a design direction in mind, begin by creating a wireframe in Figma. Use Figma’s tools to sketch a simple layout that reflects your concept. Remember that wireframing is about getting the basic structure down, so don’t worry too much about details — you can always make adjustments later!

Step 3: Components

As you start adding elements to your wireframe, create components for any reusable elements. Components in Figma work like reusable building blocks, similar to how developers create modular code. Use components for items like buttons, icons, and navigation bars to maintain consistency throughout your design.

Note: Only create components for items that will appear multiple times. If an element is unique to a single page, you don’t need to make it a component.

Step 4: Assets

With the layout coming together, add some assets to your design. You can use placeholder images, icons, or real assets if available. Adding these assets will elevate your design and make it feel more polished and professional. The closer a design is to the final product, the more useful it is for development. This type of design is often referred to as high-fidelity.

Step 5: Prototyping

Now that your design is nearly complete, add some basic interactions using Figma’s prototyping features. Focus on simple transitions, such as navigation between pages or button interactions. Aim to create one or two interactions to demonstrate the intended user flow.

Optional: If you’re feeling confident, continue working on your design and add more interactions. The Figma project you create in this lab can be a valuable asset in your portfolio, so feel free to develop it further.

Reflection

Take a moment to reflect on your experience creating this high-fidelity design in Figma:

What aspects of the design process were most challenging?

How did creating components and adding interactions enhance the usability of your design?

How might you use this design as a starting point for future projects"

ASSISTANT:
1. **Create a New Figma File**  
   - Click the “+” in the Figma dashboard to start a new file.  
   - Rename it (double-click the tab) to something like **“Landing Page Lab.”**  

2. **Set Up Your Pages**  
   - In the left sidebar, click the “+” next to **Pages** and create three pages:  
     1. **Desktop Wireframe**  
     2. **Mobile Wireframe**  
     3. **Prototype & Assets**  

3. **Build the Desktop Wireframe**  
   1. On the **Desktop Wireframe** page, press **F** and draw a Frame: set its size to **1440×1024 px**.  
   2. Rename the frame to **“Desktop – Landing.”**  
   3. **Header:**  
      - Draw a rectangle (80×40 px) top-left, add a “Logo” text layer over it.  
      - Select both, hit **⌘+Alt+K** (Mac) or **Ctrl+Alt+K** (Win) to apply Auto Layout horizontally.  
      - To the right, add four text layers (“Home,” “Features,” “About,” “Contact”), select them all and Auto Layout with even spacing.  
   4. **Hero Section:**  
      - Below header, draw a large rectangle (1200×500 px) as an image placeholder.  
      - Add a heading text (“Your Product, Elevated”) above it.  
      - Below the heading, add a 4-line body text placeholder.  
      - Draw a button: rectangle (200×50 px) + centered text (“Get Started”), group or Auto Layout it.  
   5. **Features Grid:**  
      - Draw three equal smaller frames (300×200 px) side by side using Auto Layout:  
        - Inside each, place a circle (60×60 px) for an icon placeholder, a feature title text, and a 2-line description.  
   6. **Call-To-Action:**  
      - Draw a full-width light-grey rectangle (1200×200 px).  
      - Center a heading plus a secondary button component.  
   7. **Footer:**  
      - Create a two-column Auto Layout frame:  
        - Left column: list of links as text layers.  
        - Right column: circle placeholders (24×24 px) for social icons.  

4. **Adapt for Mobile**  
   1. Switch to the **Mobile Wireframe** page.  
   2. Press **F**, create a Frame sized **375×812 px**, name it **“Mobile – Landing.”**  
   3. Repeat the same structure—header, hero, features, CTA, footer—but stacked vertically and sized for mobile.  
   4. Replace the desktop nav with a hamburger icon (24×24 px) + logo.  

5. **Create Components**  
   1. On **Prototype & Assets**, make a new frame called **“Components.”**  
   2. Inside it, build:  
      - **Button Component** (with Primary & Secondary variants via right-click → **Create component** + **Variants**)  
      - **Nav Link Component**  
      - **Feature Card Component**  
   3. Publish these to your Team Library or simply use them in your wireframes by Instance → Swap to component.  

6. **Add Placeholder Assets**  
   - In **Prototype & Assets**, draw rectangles where images will go.  
   - Select a rectangle → right-panel Fill → click the image icon → choose **“Place Image”** → leave it blank (placeholder).  

7. **Prototype Your Flow**  
   1. Duplicate your Desktop and Mobile frames, rename them to **“Desktop Prototype”** and **“Mobile Prototype.”**  
   2. Replace all buttons and nav items with component instances.  
   3. Switch to the **Prototype** tab in the right panel.  
   4. Select the “Get Started” button → drag the blue arrow → connect to the Features section on the same frame → choose **“Scroll to”** as the action.  
   5. On mobile, select the hamburger icon → connect to an overlay frame (create one) containing your menu → action **“Open overlay.”**  
   6. Set transitions to **Smart Animate** or **Dissolve** for smoothness.  

8. **Preview & Share**  
   - Click the **Play** icon in the top-right to launch the prototype preview.  
   - Interact to verify your scroll and overlay work.  
   - In preview, click **Share** → toggle **Link to prototype** → **Copy link**.  

9. **Submit Your Lab**  
   - Paste your **Figma project link** into Canvas via the **Start Assignment** button.  
   - Optionally, jot down brief reflections on what was challenging and how components/interactions helped your design.  

You now have a clean, beginner-friendly landing page wireframe with basic interactivity in Figma. Feel free to iterate by swapping in real images, refining typography, or expanding your prototype.

ASSISTANT:
1. **Download the SVG**  
   - Click the link you received and save the file (`wireframe_template.svg`) to a folder on your computer.  

2. **Open Figma and Create a New File**  
   - In Figma (desktop or web), click the **+ New file** button.  
   - Rename the file to something like **“Landing Page Template.”**  

3. **Import the SVG**  
   - In your Figma file, go to the top menu and choose **File → Import…**  
   - Navigate to your `wireframe_template.svg` and open it.  
   - Alternatively, you can simply drag-and-drop the SVG from your folder onto the Figma canvas.  

4. **Position & Rename the Imported Frame**  
   - Figma will place the SVG as a single grouped layer (often inside a Frame).  
   - In the Layers panel, double-click the top-level Frame and rename it **“Desktop Wireframe Template.”**  
   - Center it on the canvas if necessary.  

5. **Lock the Template Layer**  
   - Select the **Desktop Wireframe Template** Frame in the Layers panel.  
   - Click the lock icon next to it to prevent accidental edits.  

6. **Create Your Working Frame**  
   - With the template locked, draw a new Frame of the same size (1440×1024) on top, or simply duplicate the template Frame and rename the duplicate **“Working Wireframe.”**  
   - Unlock the duplicate so you can edit shapes, swap in real assets, or convert placeholders into components.  

7. **Convert Placeholders into Components**  
   - Select any rectangle or button shape you know you’ll reuse (e.g. the “Get Started” button).  
   - Right-click → **Create Component**.  
   - Give it a clear name (e.g. `Button / Primary`).  
   - Repeat for nav links, cards, etc.  

8. **Swap in Real Assets or Text**  
   - To replace the hero rectangle with your image: select it, then in the Fill panel click the image icon → **Choose Image** → pick your asset.  
   - To edit text placeholders, double-click the text layer in your Working Wireframe and type your copy.  

9. **Apply Auto Layout & Styles**  
   - Wrap groups of elements (e.g. header links) in Auto Layout frames: select the items → right-click → **Add Auto Layout**.  
   - Define spacing, padding, and alignment in the right-hand **Auto Layout** panel.  
   - Create shared Text and Color Styles for headings, body text, and button fills so everything stays consistent.  

10. **Build Your Prototype Page**  
   - Duplicate your Working Wireframe Frame, rename that copy **“Prototype.”**  
   - Switch to the **Prototype** tab in the right panel.  
   - Click a button → drag the node to the target frame or section → choose an interaction (e.g. “Scroll to,” “Open Overlay,” or “Navigate to”).  
   - Tweak easing or animation style (Smart Animate, Dissolve).  

11. **Preview & Share**  
   - Click the **Play ▶** icon in the toolbar to launch the interactive preview.  
   - When you’re satisfied, click **Share** → **Copy link** → paste that into Canvas for submission.  

You now have the wireframe SVG imported, locked as a template, and a working copy where you can build out your high-fidelity, interactive landing page in Figma.
