ASSISTANT:
1. Figma gives designers and developers a shared, cloud-based workspace for real-time updates, version control and a single source of truth.  
2. Developers open Figma files to inspect sizes, grab CSS snippets, export SVG/PNG assets and see exact spacing, fonts and colors.  

3. A wireframe lays out structure and content hierarchy; a prototype adds clickable flows and interactions to simulate the real experience.  
4. Early wireframing and prototyping catch layout and usability issues before code, cutting down rework and speeding up the build.  

5. A sketch wireframe is a quick, rough outline with minimal detail; a detailed wireframe adds real content, precise spacing and basic styling.  
6. Teams pick low-fidelity wireframes when they need fast feedback on structure or when exploring multiple ideas without investing in visuals.  

7. High-fidelity designs should include final typography, color palette, component states, spacing specs, asset exports and responsive breakpoints.  
8. Figma spits out measurements, CSS code and downloadable assets so developers can translate design elements into code pixel-perfectly.  

9. Design systems in Figma bundle components, styles and tokens into libraries that enforce consistent buttons, forms, colors and typography across the project.  
10. Reusable components in both design tools and code reduce duplication, speed up updates and keep the user interface cohesive.  

11. Figma libraries map directly to frontend frameworks—think React components or Tailwind utility classes—so design tokens become code patterns.  
12. Teams use frameworks like React or Tailwind CSS to leverage prebuilt, responsive components and utility classes instead of writing custom CSS for every element.  

13. Well-designed apps like Airbnb or Google Docs rely on Figma-driven design systems to keep UI patterns consistent as they scale.  
14. Google’s Material Design, Apple’s Human Interface Guidelines and Airbnb’s Design Language System all live in Figma libraries, letting designers and developers stay in sync on spacing, typography and component behavior.

# Step 1: Project Structure & File Setup

1. **Create your project folder**  
   - Inside it, add two files:  
     - `index.html`  
     - `styles.css`

2. **Open `index.html` in your code editor** and add this basic skeleton:

   ```html
   <!DOCTYPE html>
   <html lang="en">
   <head>
     <meta charset="UTF-8" />
     <meta name="viewport" content="width=device-width, initial-scale=1.0" />
     <!-- Google Fonts: Rubik -->
     <link
       href="https://fonts.googleapis.com/css2?family=Rubik:wght@400;500;700&display=swap"
       rel="stylesheet"
     />
     <!-- Main stylesheet -->
     <link rel="stylesheet" href="styles.css" />
     <title>Simple Booking Section</title>
   </head>
   <body>
     <!-- We’ll add our “Simple booking” section here in Step 4 -->
   </body>
   </html>
   ```

   **Explanation:**  
   - `<!DOCTYPE html>`: tells the browser we’re using HTML5.  
   - `<meta charset="UTF-8">`: ensures text (like “Simple booking”) displays correctly.  
   - `<meta name="viewport">`: makes the page scale nicely on mobile devices.  
   - Google Fonts link: pulls in the **Rubik** font family with weights 400, 500, 700.  
   - `styles.css`: connects our custom CSS.  

3. **Open `styles.css`** (it can be empty for now). We’ll populate it in Step 3.

Once these files exist and are linked, you can open `index.html` in your browser—nothing visible yet, but no errors in the console means we’re set.
