## 2. Build `ColorInput.tsx`

```tsx
// src/components/ColorInput/ColorInput.tsx

import React from 'react';

interface ColorInputProps {
  onColorChange: (color: string) => void;
}

export const ColorInput: React.FC<ColorInputProps> = ({ onColorChange }) => {
  // When the user types, capture the new value and pass it up
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    onColorChange(e.target.value);
  };

  return (
    <input
      type="text"
      placeholder="Enter a color"
      onChange={handleChange}
      style={{ padding: '0.5rem', width: '100%' }}
    />
  );
};
```

- **Why this code?**  
  1. **Props interface** (`ColorInputProps`) declares exactly what the parent must provide: a function receiving a `string`.  
  2. **`handleChange`** extracts `e.target.value` (the current input) and calls the parentвҖҷs callback.  
  3. Using `placeholder` and inline styles makes the input user-friendly immediately.

---

## 3. Build `ColorPicker.tsx`

```tsx
// src/components/ColorPicker/ColorPicker.tsx

import React, { useState } from 'react';
import { ColorInput } from '../ColorInput/ColorInput';

export const ColorPicker: React.FC = () => {
  // 1) Declare a piece of state to hold the current color
  const [color, setColor] = useState('white');

  // 2) Handler to update state when child reports a new color
  const handleColorChange = (newColor: string) => {
    setColor(newColor);
  };

  return (
    <div style={{ fontFamily: 'sans-serif', maxWidth: 400, margin: '0 auto' }}>
      <h2>Color Picker</h2>
      {/* Pass our handler down to the child */}
      <ColorInput onColorChange={handleColorChange} />

      {/* Visual feedback: box whose background matches the chosen color */}
      <div
        style={{
          marginTop: '1rem',
          padding: '1rem',
          backgroundColor: color,
          color: color === 'black' ? 'white' : 'black',
          textAlign: 'center',
          border: '1px solid #ccc',
        }}
      >
        Current color: {color}
      </div>
    </div>
  );
};
```

- **Why this code?**  
  1. **`useState`** holds our live `color` value.  
  2. **`handleColorChange`** is the callback passed to the child; it receives the new color and updates state.  
  3. The `<div>` below uses `backgroundColor: color` so you immediately see the effect of typing вҖңblueвҖқ, вҖң#ff0000вҖқ, etc.  
  4. A bit of conditional styling (`color === 'black'`) ensures text remains legible.

## 4. Integrate in your app

```tsx
// src/App.tsx

import React from 'react';
import { ColorPicker } from './components/ColorPicker/ColorPicker';

function App() {
  return (
    <div style={{ padding: '2rem' }}>
      <ColorPicker />
    </div>
  );
}

export default App;
```

- **Why?** This is the simplest mounting point; it keeps your components decoupled and focused.

---

## 5. How it all works together

1. **User types** in `ColorInput`.  
2. **`handleChange`** in the child grabs the new text and calls `onColorChange(newValue)`.  
3. **ParentвҖҷs `handleColorChange`** runs, calling `setColor(newValue)`.  
4. **React re-renders** `ColorPicker`, updating the colored `<div>`.  

This demonstrates clear **parent-child communication**, **state management**, and **dynamic styling**.
