# Top task (now): make `@/components/*` resolve on Netlify

## 1) Pick the **correct alias** based on your folder layout

**If your components are under `src/components/`** (most common):
```json
// tsconfig.json  (or jsconfig.json)
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@/*": ["src/*"] }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

**If your components are at repo root `components/`**:
```json
// tsconfig.json  (or jsconfig.json)
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@/*": ["*"] }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

> Keep only **one** of those mappings. Commit this file at repo root.

## 2) Ensure files exist, are tracked, and exports match imports

- Confirm each file exists:
  - `src/components/CookieBanner.tsx`
  - `src/components/Header.tsx`
  - `src/components/Footer.tsx`
  - `src/components/SeoSchema.tsx`
  - `src/components/Hero.tsx`
  - …or the `components/` equivalents if not using `src/`.

- If you import without braces (e.g., `import CookieBanner from '@/components/CookieBanner'`), the file must have a **default export**:
  ```ts
  export default function CookieBanner() { /* ... */ }
  ```

- Make sure no `.gitignore` rule accidentally excludes `components/` or `src/`.

## 3) Fix **Linux case sensitivity** (common Netlify failure)
On Windows/macOS it can appear fine; Netlify (Linux) is strict. Match import case to filenames exactly.

Example: file is `CookieBanner.tsx` but you imported `'@/components/cookiebanner'` → build fails.

If you need a case-only rename that Git will notice:
```bash
git config core.ignorecase false
git mv src/components/cookiebanner.tsx src/components/CookieBanner.tsx
git commit -m "fix: correct component filename case for Linux builds"
```

## 4) Local precheck before pushing
Run locally and fix any remaining unresolved imports:
```bash
npm run build  # or: npx next build
```

## 5) Netlify sanity
- Node 20 is fine (your log shows v20.19.4).  
- Build command: `next build` (or your package.json’s `build` script).  
- If you use `netlify.toml`, keep it simple:
```toml
[build]
command = "next build"
publish = ".next"
```
(Netlify’s Next adapter may override `publish`; the critical part is that the **build succeeds** with your aliases.)
