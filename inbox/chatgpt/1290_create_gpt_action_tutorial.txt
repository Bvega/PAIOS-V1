How to create a custom GPT action

--------------------------------------------------------------------------------

ASSISTANT:
Creating a custom GPT action involves several steps to define the action's purpose, develop the script or code, and integrate it into the GPT framework. Here's a detailed guide to help you through the process:

### Step-by-Step Guide to Creating a Custom GPT Action

1. **Define the Purpose:**
   - Clearly articulate what the custom action is supposed to do. For example, "fetching weather information," "generating reports," or "sending email notifications."

2. **Outline Requirements:**
   - List all necessary inputs, processes, and outputs.
   - Identify any external APIs, libraries, or tools required.

3. **Script Development:**
   - Choose a programming language that fits well within the GPT environment (e.g., Python, JavaScript).
   - Write the script to perform the desired action. Here’s an example of a simple Python script for fetching weather data from an API:

   ```python
   import requests

   def fetch_weather(api_key, location):
       url = f"http://api.weatherapi.com/v1/current.json?key={api_key}&q={location}"
       response = requests.get(url)
       if response.status_code == 200:
           return response.json()
       else:
           return {"error": "Unable to fetch weather data"}

   # Example usage
   api_key = "your_api_key_here"
   location = "San Francisco"
   weather_data = fetch_weather(api_key, location)
   print(weather_data)
   ```

4. **Integrate with GPT Framework:**
   - Determine how to call this action within the GPT environment. This could be a direct function call, a webhook, or an API endpoint.
   - Ensure the script is accessible and executable by the GPT instance.

5. **Testing:**
   - Run various test cases to ensure the action performs as expected.
   - Handle edge cases and error conditions gracefully.

6. **Deployment:**
   - Deploy the action within your GPT setup.
   - Ensure it’s well-documented and maintainable.

7. **Documentation:**
   - Create comprehensive documentation detailing how to use the custom action, including code snippets, examples, and troubleshooting tips.

### Example: Custom Action to Fetch Weather Data

Here’s a complete example from definition to deployment:

**Purpose:** Fetch current weather data for a given location.

**Requirements:**
- Input: API key, location name
- Process: Call weather API, process response
- Output: Current weather data

**Script Development:**

```python
import requests

def fetch_weather(api_key, location):
    url = f"http://api.weatherapi.com/v1/current.json?key={api_key}&q={location}"
    response = requests.get(url)
    if response.status_code == 200:
        return response.json()
    else:
        return {"error": "Unable to fetch weather data"}
```

**Integration with GPT Framework:**

Assuming the GPT environment allows direct execution of Python scripts:

```python
# GPT Framework Integration Example

def gpt_custom_action(api_key, location):
    weather_data = fetch_weather(api_key, location)
    return weather_data
```

**Testing:**

```python
def test_fetch_weather():
    api_key = "valid_api_key"
    location = "San Francisco"
    result = fetch_weather(api_key, location)
    assert "current" in result, "Test failed: 'current' key not found in result"

test_fetch_weather()
print("All tests passed.")
```

**Deployment:**

Deploy the action by uploading the script to the GPT’s environment and ensure it’s callable.

**Documentation:**

- **Usage:** Provide clear instructions on how to call the action.
- **Example Call:**

```python
api_key = "your_api_key"
location = "San Francisco"
print(gpt_custom_action(api_key, location))
```

By following these steps, you can create custom actions tailored to your specific needs within a GPT framework. If you have any specific requirements or need help with a particular part of the process, feel free to ask!
