## 4) Learning System → Tracking & Improvement
**Telemetry schema (per engagement)**
- `client_profile`: size, stack, practice mix, constraints  
- `persona_DNA`: list of skills/tactics chosen (IDs), weights  
- `plan_quality_scores`: rubric 1–5 for Fit, Security, Feasibility, Adoption, Clarity  
- `engagement_outcomes`: adoption %, cycle time, incidents, rework count  
- `user_feedback`: attorney/staff ratings, qualitative notes  
- `ab_tests`: alternative persona mixes tried, winner, lift %  
- `versions`: persona template version, prompt version, LLM model, date

**Loop**
- Log all runs → score with human + rule checks → retrain skill weighting  
- Use multi-armed bandit to pick among top persona mixes for similar clients  
- Cluster past clients by vectorized requirements → retrieve best-performing DNA

## 5) Scalability → Hundreds of Expert Combinations
- **Composable ontology:** Maintain a library of atomic skills/constraints (“skills graph”).  
- **Retrieval:** Embed client brief → retrieve top skills by similarity + rules (e.g., “law firm” ⇒ legal hold).  
- **Assembler:** Score/weight skills; generate persona object; enforce governance (security-first skills always included).  
- **Caching:** Memoize frequent combos; snapshot “gold personas” per segment.  
- **Evaluation at scale:** Batch rubric scoring + anomaly checks; nightly leaderboards.  
- **Governance:** Versioned schemas, change logs, red-team prompts, PII scrubbing.

**Minimal schema (TypeScript)**
```ts
type SkillID = string;

interface PersonaDNA {
  roleTitle: string;
  mission: string;
  skills: { id: SkillID; weight: number }[];
  guardrails: string[];           // must include RBAC/SSO/etc. where relevant
  discoveryQuestions: string[];
  kpis: string[];
  playbook: string[];             // high-level steps
  assumptions: string[];
  version: string;                // e.g., dna-2025.08
}
```

## 6) Limitations & Challenges (with mitigations)
- **Hallucinations / over-generalization:** bind outputs to schemas, checklists, and rule validators; require sources when feasible.  
- **Privacy & legal ethics:** strict PII scrubbing; tenant isolation; minimize client data in prompts; access controls.  
- **Evaluation noise:** combine human rubric + outcome metrics; use control groups.  
- **Prompt drift/versioning:** lock prompts by version; run regression tests.  
- **Bias / vendor lock-in:** generic patterns by default; vendor picks only on explicit criteria.  
- **Cost/latency at scale:** caching, smaller models for retrieval, batch scoring.
