Ensure each step aligns with the lab’s goals: minimal custom CSS, Bootstrap utilities/components, and responsive design via the grid system.

   - Confirm your original custom stylesheet (`styles.css`) is commented out or removed.

2. **Wrap in a Bootstrap Grid Container**  
   - Replace your top-level `<div class="card">…</div>` with:
     ```html
     <div class="container d-flex justify-content-center align-items-center vh-100">
       <!-- Card goes here -->
     </div>
     ```
   - This uses Bootstrap’s **container**, **flex utilities**, and full-height viewport class.

3. **Use the Bootstrap Card Component**  
   - Inside that container, wrap your content in:
     ```html
     <div class="card text-center">
       <!-- Image + body go here -->
     </div>
     ```
   - This replaces any custom “card__” wrappers.

4. **Replace the Image Wrapper**  
   - Remove extra `<div class="card__image">`.  
   - Use:
     ```html
     <img
       src="images/image-qr-code.png"
       alt="QR Code"
       class="card-img-top img-fluid p-3"
     />
     ```
   - **card-img-top** and **img-fluid** ensure proper Bootstrap styling and responsiveness; **p-3** adds padding.

5. **Move Text into `.card-body`**  
   - Group your heading and paragraph inside:
     ```html
     <div class="card-body">
       <h1 class="card-title mb-3 fs-5 fw-bold">…</h1>
       <p class="card-text text-muted mb-0">…</p>
     </div>
     ```
   - Use **mb-3**, **fs-5**, **fw-bold**, **text-muted**, and **mb-0** to handle spacing, font-size, weight, and color.

   - Delete or fully remove any `styles.css` file and all `.card__*` class definitions.
   - Rely exclusively on Bootstrap utility classes and components.

7. **Ensure Responsive Centering**  
   - Confirm the flex container remains centered at **all** breakpoints.  
   - Optionally, you can swap `vh-100` for `min-vh-100` if you’d like more flexible vertical centering.

   git push

**Quick Guide: Forking Your Own GitHub Repo**

GitHub won’t let you “Fork” a repo into the same account, so you can use the **Import Repository** feature to make an independent copy under your user. Follow these steps:

1. **Open the Import Tool**  
   - In your browser, go to:  
     ```
     https://github.com/new/import
     ```

2. **Enter the Original Repo URL**  
   - In **“Your old repository’s clone URL”**, paste the full HTTPS URL of your repo, e.g.:  
     ```
     https://github.com/Bvega/blog-preview-card.git
     ```

3. **Choose the Owner**  
   - In **“Owner”**, select your username (e.g. `Bvega`).

4. **Pick a New Name**  
   - In **“Name”**, give your fork a distinct name, such as `blog-preview-card-fork`.

5. **Begin Import**  
   - Click **“Begin import”**.  
   - Wait a minute while GitHub copies the code into your new repo.

6. **Verify the Fork**  
   - Navigate to **Your repositories** (`https://github.com/Bvega?tab=repositories`)  
   - You’ll see `blog-preview-card-fork` alongside your originals.

---

**Alternative (GitHub CLI)**  
If you have the GitHub CLI installed, you can run:
```bash
gh repo fork Bvega/blog-preview-card --clone=false --remote=false --org=YourOrgOrUsername
```
Then rename or clone locally as needed.

---

2. Run **Git: Clone**.  
3. Pick **GitHub** when prompted, then select your newly forked repo (`YourUsername/YourRepo`).  
4. Choose a local folder—VS Code will clone it and open the workspace for you.

---
