ASSISTANT:
1. **Why use semantic HTML like `<header>`, `<article>`, `<footer>` instead of `<div>`?**  
Because they tell you what the content is about. It's like labeling your boxes — so you (and search engines or screen readers) understand what's inside.

2. **How does semantic HTML help with readability and accessibility?**  
It makes the code easier to read and helps tools like screen readers explain the page better to users with disabilities.

3. **Flexbox vs. CSS Grid – what’s the difference and when to use them?**  
- Flexbox is best for arranging items **in one direction** (row or column).  
- Grid is better when you need to arrange things **in two directions** (rows and columns).  
Use Flexbox for simple layouts; use Grid for more complex page layouts.

4. **How does the CSS box model affect layout?**  
Everything on the page is a box. Padding, border, and margin decide how big the box is and how it sits next to other boxes. Misunderstanding this can break your layout.

5. **How to make a webpage responsive for all screens?**  
- Use **media queries** to adjust styles for different screen sizes.  
- Use **flexible layouts** like Flexbox or Grid.  
- Use **relative units** (%, em, rem) instead of fixed units (px).  
- Make images and text scale to fit the screen.
