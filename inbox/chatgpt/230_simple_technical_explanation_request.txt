1. **What are the differences between var, let, and const, and when would you use each?**  
   - **var** is function-scoped and allows redeclaration; it can lead to unexpected behavior due to hoisting.  
   - **let** is block-scoped, cannot be redeclared in the same scope, and helps avoid accidental variable collisions.  
   - **const** is block-scoped like let but also signals that the binding won’t change; use it for values you don’t intend to reassign.  

2. **How do function parameters improve code reusability?**  
   - Function parameters let you pass different values into the same function logic. By avoiding hard-coded values, you can call the same function with varying inputs, reducing duplication and making your code modular.  

3. **How can using if...else or switch statements make your code more dynamic and interactive?**  
   - They let your program choose different execution paths based on conditions. This branching allows your app to respond to user input, data states, or other criteria, producing varied behavior without rewriting code.  

4. **What are some advantages of using logical operators like && and || in conditional expressions?**  
   - **&&** lets you require multiple conditions to be true before proceeding, simplifying nested ifs.  
   - **||** lets you provide fallback values or take action when at least one condition is true. Both make conditions concise and readable.  

5. **What are the key differences between for, while, and do...while loops? When might you choose one over the other?**  
   - **for** loops are ideal when you know how many iterations you need up front (e.g., iterating over an array index).  
   - **while** loops run as long as a condition remains true and check the condition before each iteration. Use them when the number of iterations isn’t predetermined.  
   - **do...while** loops run once before checking the condition, ensuring the loop body always executes at least once. Use this when you need a guaranteed first pass.  

6. **How might the skills reviewed in this module (e.g., functions, loops) be used to build a small web application?**  
   - You’d break tasks into functions (e.g., data fetch, rendering UI, handling input), use loops to process lists of items (like menu entries or form fields), and use conditionals to update the interface based on user actions or data states—creating an organized, interactive app.  

7. **What are some challenges you have faced when writing JavaScript, and how did you overcome them?**  
   - Common challenges include tracking asynchronous operations, managing variable scope, and avoiding deeply nested callbacks. Overcoming them involved using promises/async-await for async flows, choosing let/const for clear scoping, and breaking logic into smaller functions.  

8. **How does writing clean, reusable JavaScript code benefit you and your team in the long term?**  
   - Clean, reusable code is easier to maintain, debug, and extend. It reduces duplication, speeds onboarding of new team members, and lowers the risk of bugs when adding features or fixing issues.
