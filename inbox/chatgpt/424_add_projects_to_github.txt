ASSISTANT:
To add **different projects** to your **GitHub repository**, you have a few options depending on how you want to organize them. Here are the most common methods:

---

### ✅ OPTION 1: **Multiple Projects in One Repository (Monorepo Style)**

Ideal if your projects are **related** or you're managing them as a collection (like frontend/backend, modules, etc.)

#### Steps:

1. **On your local machine**, create a directory:
   ```bash
   mkdir my-repo
   cd my-repo
   ```

2. **Create folders for each project inside**:
   ```bash
   mkdir project1 project2 project3
   ```

3. **Initialize Git (if not done already)**:
   ```bash
   git init
   ```

4. **Add your project files into each folder**.

5. **Stage and commit**:
   ```bash
   git add .
   git commit -m "Add multiple projects"
   ```

6. **Push to GitHub**:
   ```bash
   git remote add origin https://github.com/username/repo-name.git
   git push -u origin main
   ```

---

### ✅ OPTION 2: **Separate Repositories for Each Project**

Best when the projects are **independent**.

#### Steps:
1. **Create a separate GitHub repository for each project** from GitHub UI.

2. **For each project locally**:
   ```bash
   cd my-project-folder
   git init
   git remote add origin https://github.com/username/project-repo.git
   git add .
   git commit -m "Initial commit"
   git push -u origin main
   ```

---

### ✅ OPTION 3: **Use Git Submodules**

Useful if your main repo depends on other repos.

#### Steps:
1. **Inside your main project repo**, run:
   ```bash
   git submodule add https://github.com/username/project1.git projects/project1
   git submodule add https://github.com/username/project2.git projects/project2
   ```

2. It adds a `.gitmodules` file to track submodules.

3. Commit and push:
   ```bash
   git add .gitmodules projects/
   git commit -m "Add submodules"
   git push
   ```

ASSISTANT:
To **add another repository in GitHub**, follow this step-by-step guide depending on your situation:

---

## 🔹 CASE 1: **Create a New GitHub Repository and Push Code**
Use this when you have a new local project you want to push to GitHub as a new repo.

### ✅ Step-by-Step:

1. **Go to GitHub** → https://github.com → Click **"New repository"**

2. **Fill in details**:
   - Repository name (e.g. `project-2`)
   - Public or Private
   - *Don’t* initialize with README (you already have code locally)

3. **Create repo**, GitHub gives you commands. Follow these:

```bash
cd path/to/your/project-folder
git init                     # if not already a git repo
git remote add origin https://github.com/yourusername/project-2.git
git add .
git commit -m "Initial commit"
git push -u origin main
```

---

## 🔹 CASE 2: **Add a New Folder as Another Repo Inside a Main Repo Folder**

This is **not recommended** usually but can be done by nesting repos (also known as sub-repos).

> Use this only if you understand nested repos or want to experiment.

```bash
cd main-repo-folder
mkdir new-project
cd new-project
git init
git remote add origin https://github.com/yourusername/new-project.git
```

⚠️ Be careful: nested repos inside Git folders are ignored by the parent Git repo unless added as **submodules** (see next case).

---

## 🔹 CASE 3: **Add Another GitHub Repo as a Submodule**

Use this if you want your main repo to track another repo (as a folder).

```bash
cd your-main-project
git submodule add https://github.com/yourusername/other-repo.git path/to/folder
git commit -m "Add submodule"
git push
```

---

### 🔄 TL;DR
- **Want new GitHub repo?** → Create it on GitHub → Push from local.
- **Want to include another repo in your main repo?** → Use submodule.
