Here are the **main basic knowledge concepts** of React, based on content from _"The Road to React"_ and _"Learn React with TypeScript"_:

### 🧠 Core React Concepts

1. **JSX (JavaScript XML)**:
   - JSX allows writing HTML-like syntax within JavaScript.
   - It's syntactic sugar for `React.createElement()`.

2. **Components**:
   - Fundamental building blocks of a React application.
   - Two main types: **Function Components** and **Class Components**.
   - Reusable and composable.

3. **Props**:
   - Short for “properties”.
   - Inputs to components; passed from parent to child.
   - Immutable within the receiving component.

4. **State**:
   - Data that changes over time.
   - Managed within components using `useState` (in function components).

5. **Events**:
   - Handlers like `onClick`, `onChange` used to respond to user actions.
   - Defined as functions and passed to elements.

6. **Hooks**:
   - Functions that let you use state and other React features in function components.
   - Most common: `useState`, `useEffect`.

7. **useEffect Hook**:
   - Handles side effects like data fetching, subscriptions, etc.
   - Runs after rendering and can be configured to re-run on specific data changes.

8. **Component Tree**:
   - React apps are a hierarchy of nested components.
   - Data flows **top-down** from parent to child via props.

9. **Virtual DOM**:
   - React maintains a lightweight copy of the DOM.
   - Efficient diffing algorithm updates only what’s changed in the actual DOM.

10. **Keys in Lists**:
   - Unique identifiers help React optimize rendering when working with lists.
   - Typically used in `.map()` rendering of components.

11. **Controlled vs Uncontrolled Components**:
   - Controlled: Form inputs tied to state.
   - Uncontrolled: Managed via `refs`.

12. **React Dev Tools**:
   - Browser extension to inspect and debug React component trees.
