**Learning Goals (plain):**

**Step-by-Step (choose your path):**

**A) Voice Path (phone/tablet)**  
1. Open the AI app (or website).  
2. Find and tap the **microphone** icon.  
3. Say: “Hi, I’m new. Please speak slowly. What can you help with?”  
4. When it replies, say: “Explain using everyday examples.”  
5. Try a real task: “Plan a simple dinner for two using chicken and rice.”  
6. Follow-up: “Make a shopping list with prices.”  
7. Safety check: “Before I use this, what personal info should I **never** share with AI?”

**B) Text Path (computer or tablet/phone keyboard)**  
1. Click in the message box.  
2. Type: “I’m brand new. Explain what you can do in 3 sentences.”  
3. Task: “Summarize this paragraph I paste next.” (Paste a small news paragraph.)  
4. Follow-up: “Rewrite it as a friendly email to my book club.”  
5. Safety check: “List 3 things I should never type into an AI chat.”

**Practice Coach—Mini Exercises (print-friendly):**  
- Try 3 prompts from daily life:  
  1) “Draft a thank-you note to my neighbor for watering plants.”  
  2) “Give me 5 gentle ways to say I can’t attend an event.”  
  3) “Explain my phone’s storage alert in simple steps.”  
- Confidence scale (0–5): How confident do you feel using AI for small tasks?

**Troubleshooting (top snags):**  
- Can’t find the mic: update the app; check Settings → Privacy → Microphone → enable.  
- AI speaks too fast: ask “Please speak more slowly.”  
- Overwhelmed by text: say “Use short bullet points.”

**Accessibility Notes:**  
- Large-print handout (20 pt), one column.  
- Screenshot callouts with arrows and labels.  
- Provide audio version (read aloud of lesson).

**Wrap-Up Prompt:**

# Core Concept: “Agent Foundry GPT”
A single GPT that:
1) **Interprets the task** → 2) **Selects/creates agents** → 3) **Runs a short agent round** (debate or pipeline) → 4) **Merges outputs** into the user’s requested deliverable(s) → 5) **Presents next-step options**.

---

# 1) System Instructions (paste into your Custom GPT)
Use this as your **Instructions** block. It makes the GPT self-organize the right agents for any task.

> You are **Agent Foundry**: an on-demand multi-agent orchestrator.  
> **Mission:** When a user states a goal or task, you (a) clarify only if essential, (b) infer the work domain, (c) generate the minimum set of specialized “agents” with crisp role cards, (d) run a brief agent round (sequence or round-table), and (e) produce the final deliverables.  
> **Default style:** concise, plain language, action-first.  
> **Never say you will do work later—return your best complete draft now.**
>
> ### Workflow
> 1) **Scope & Constraints:** Extract task, target outcome, constraints (time, format, tone, audience).  
> 2) **Agent Selection:** Instantiate 3–6 agents from a flexible taxonomy (below) or mint custom ones; each gets a one-sentence charter + 3 responsibilities.  
> 3) **Collaboration Mode:**  
>    - *Pipeline* (ordered handoff) for structured tasks (docs, plans, briefs).  
>    - *Round-table* (short debate) for open problems; resolve conflicts with a decision rule.  
> 4) **Synthesis:** Merge agent outputs into the requested deliverable(s).  
> 5) **QA Pass:** Sanity check; list assumptions; provide next actions.  
> 6) **Packaged Output:** Clear headings, checklists, and a concise summary.
>
> ### Agent Taxonomy (pick & adapt)
> - **Strategist** (objectives, constraints, tradeoffs)  
> - **Researcher** (fact-finding; cites sources when browsing is enabled)  
> - **Creator** (writes/makes: content, designs, prompts)  
> - **Engineer/Implementer** (step-by-step builds, pseudo-code, SOPs)  
> - **UX/Comms** (audience fit, tone, formatting)  
> - **Risk & Quality** (edge cases, ethical/legal/safety checks)  
> - **Analyst** (metrics, success criteria, cost/time estimates)  
> - **Operator** (project plan, roles, timeline, RACI)
>
> ### Decision Rules
> - **Minimality:** Use the fewest agents that cover the task.

# Using a Custom GPT to Call Another LLM

## Core Limitation

* Custom GPTs **cannot directly call another LLM** inside the GPT Builder environment.
* They only support:

  * Knowledge retrieval (files, connected sources)
  * Built-in tools (web, gmail, gcal, etc.)
  * Custom **function calls** (to APIs you define)

## Indirect Methods

### 1. API Gateway Approach

* Host the other LLM (Claude, Gemini, LLaMA, or even another OpenAI model) behind an API endpoint.
* Add that API as a **function** in the GPT Builder.
* Flow: Custom GPT → API function → Forwarded request to target LLM → Response returned to GPT.

### 2. Orchestrator Layer

* Set up a middleware/orchestration service (FastAPI, Node.js, LangChain, etc.).
* Custom GPT calls this orchestrator via function call.
* Orchestrator decides:

  * Which LLM to use.
  * How to format input/output.
  * Whether to chain multiple models together.

## Not Possible

* No native “swap model” option in GPT Builder.
* Cannot directly integrate other LLMs without API indirection.

## Practical Outcome

* With function calling, you can simulate "multi-LLM" behavior:

  * GPT delegates tasks to your API.
  * Your API manages multiple LLMs in the background.

# Orchestrator sketch (FastAPI + adapters)

```python
# fastapi_app.py
from fastapi import FastAPI
from pydantic import BaseModel
from typing import List, Optional, Dict, Any
from adapters import openai_call, anthropic_call, gemini_call

app = FastAPI()

class Constraints(BaseModel):
    deadline: Optional[str] = None
    audience: Optional[str] = None
    tone: Optional[str] = None
    budget: Optional[str] = None
    tools: Optional[List[str]] = None

class OrchestratePayload(BaseModel):
    task: str
    deliverables: Optional[List[str]] = None
    constraints: Optional[Constraints] = None
    notes: Optional[str] = None

@app.post("/orchestrate_task")
def orchestrate(payload: OrchestratePayload) -> Dict[str, Any]:
    # 1) Intent & domain
    domain = detect_domain(payload.task, payload.notes)

    # 2) Agent selection (3–6 agents)
    roster = create_agents(domain, payload)

    # 3) Plan (pipeline or round-table)
    plan = plan_collab(roster, payload)

    # 4) Execute (route per step)
    traces, partials = [], []
    for step in plan:
        model_target = route(step, domain, payload.constraints)
        if model_target == "openai":
            out = openai_call(step.prompt)
        elif model_target == "anthropic":
            out = anthropic_call(step.prompt)
        elif model_target == "gemini":
            out = gemini_call(step.prompt)
        else:
            out = openai_call(step.prompt)  # default
        traces.append({"agent": step.agent, "model": model_target})
        partials.append({"agent": step.agent, "output": out})

    # 5) Synthesis + QA
    deliverables = synthesize(partials, payload.deliverables)
    risks, next_steps = assess_risk(deliverables, domain), propose_next_steps(payload)

    return {
        "task_snapshot": make_snapshot(payload, domain),
        "agent_roster": [a.dict() for a in roster],
        "collab_log": [t for t in summarize_traces(traces)],
        "deliverables": deliverables,
        "risks": risks,
        "next_steps": next_steps
    }
```

## Why multiple repos? (Poly-repo strategy)

We picked **polyrepo** as the best practice earlier. That means:  
- **One repo = one responsibility**  
- Each repo can have its own CI/CD, versioning, and release cycle  
- Cleaner for contributors: you don’t need to download the whole project just to fix docs or an adapter  

### Example
- `agent-foundry-orchestrator` → the FastAPI backend service  
- `agent-foundry-adapters` → connectors to OpenAI, Anthropic, Gemini (can be published as a pip package)  
- `agent-foundry-agents` → the role cards and prompt patterns  
- `agent-foundry-gpt-config` → the JSON schema and GPT instructions (just text + docs)  
- `agent-foundry-docs` → the documentation site (mkdocs/Docusaurus)  
- `agent-foundry-tooling` → Postman collection, test harness, infra scripts  

So if someone only wants to contribute new **agents** or **docs**, they don’t have to clone the orchestrator or adapters.  

---

## What if I don’t want so many repos?
You absolutely don’t *have* to.  
- You could use a **monorepo** (everything in one repo under folders). Easier to start, but heavier to maintain later.  
- Or you could do a **hybrid**: start monorepo for speed, split into polyrepo later if it grows.

from fastapi import FastAPI
from pydantic import BaseModel
from typing import List, Optional, Dict, Any

app = FastAPI()

@app.get("/healthz")
def healthz():
    return {"status": "ok"}

class Constraints(BaseModel):
    deadline: Optional[str] = None
    audience: Optional[str] = None
    tone: Optional[str] = None
    budget: Optional[str] = None
    tools: Optional[List[str]] = None

class OrchestratePayload(BaseModel):
    task: str
    deliverables: Optional[List[str]] = None
    constraints: Optional[Constraints] = None
    notes: Optional[str] = None

@app.post("/orchestrate_task")
def orchestrate_task(payload: OrchestratePayload) -> Dict[str, Any]:
    roster = [
        {"name": "Strategist", "charter": "Set objectives", "duties": ["scope", "tradeoffs", "constraints"]},
        {"name": "Creator", "charter": "Draft deliverables", "duties": ["outline", "write", "polish"]},
        {"name": "QA", "charter": "Sanity-check", "duties": ["risks", "edits", "next steps"]},
    ]
    return {
        "task_snapshot": {
            "task": payload.task,
            "deliverables": payload.deliverables or [],
            "assumptions": ["MVP stub – replace with planner in M2"]
        },
        "agent_roster": roster,
        "collab_log": ["Chose pipeline mode (simple stub)"],
        "deliverables": {
            "primary": f"# Draft for: {payload.task}\n\n(Replace with real synthesis in M2)"
        },
        "risks": ["Early stub; outputs incomplete"],
        "next_steps": ["Implement planner", "Implement synthesizer", "Add adapters"]
    }

## 2) Update the API to load roles.yaml and produce a simple deliverable

Replace the file contents with this version (adds YAML loading and a tiny synthesizer):

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List, Optional, Dict, Any
import os
import json

# NEW: yaml loader
try:
    import yaml  # type: ignore
except ImportError:
    yaml = None

app = FastAPI()

@app.get("/healthz")
def healthz():
    return {"status": "ok"}

# ---------------------------
# Models
# ---------------------------
class Constraints(BaseModel):
    deadline: Optional[str] = None
    audience: Optional[str] = None
    tone: Optional[str] = None
    budget: Optional[str] = None
    tools: Optional[List[str]] = None

class OrchestratePayload(BaseModel):
    task: str
    deliverables: Optional[List[str]] = None
    constraints: Optional[Constraints] = None
    notes: Optional[str] = None

# ---------------------------
# Helpers
# ---------------------------
def load_roles() -> List[Dict[str, Any]]:
    """
    M2: load roles from a vendored file (data/roles.yaml).
    M6: replace with a proper dependency on agent-foundry-agents.
    """
    roles_path = os.path.join(os.path.dirname(__file__), "data", "roles.yaml")
    if not os.path.exists(roles_path):
        raise FileNotFoundError(f"roles.yaml not found at {roles_path}")

    if yaml is None:
        # Fallback: provide minimal roles if PyYAML missing
        return [
            {"name": "Strategist", "charter": "Set objectives", "duties": ["scope", "tradeoffs", "constraints"]},
            {"name": "Creator", "charter": "Draft deliverables", "duties": ["outline", "write", "polish"]},
            {"name": "QA", "charter": "Sanity-check", "duties": ["risks", "edits", "next steps"]},
        ]

    with open(roles_path, "r", encoding="utf-8") as f:
        data = yaml.safe_load(f) or []
    # basic shape validation
    roles = []
    for r in data:
        roles.append({
            "name": r.get("name", "Unknown"),
            "charter": r.get("charter", ""),
            "duties": r.get("duties", []),
        })
    return roles

def pick_mode(deliverables: Optional[List[str]]) -> str:
    """
    Tiny heuristic: if user wants a plan/brief, use pipeline, else round-table.
    """
    if not deliverables:
        return "round-table"
    text = " ".join(deliverables).lower()
    if any(k in text for k in ["plan", "brief", "checklist", "outline"]):
        return "pipeline"
    return "round-table"

def synthesize(task: str, roster: List[Dict[str, Any]], mode: str) -> str:
    """
    Very small M2 synthesizer: produce a short markdown draft.
    """
    roles_line = ", ".join([r["name"] for r in roster])
    return (
        f"# Draft for: {task}\n\n"
        f"**Mode:** {mode}\n\n"
        f"**Agents:** {roles_line}\n\n"
        f"## Summary\n"
        f"- This is an M2 stub synthesis. In later milestones we’ll call real LLMs and merge.\n"
        f"- Next we’ll add adapters and smarter planning.\n"
    )

# ---------------------------
# Routes
# ---------------------------
@app.post("/orchestrate_task")
def orchestrate_task(payload: OrchestratePayload) -> Dict[str, Any]:
    try:
        roster = load_roles()
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"Failed to load roles: {e}")

    mode = pick_mode(payload.deliverables)
    primary_md = synthesize(payload.task, roster, mode)

    return {
        "task_snapshot": {
            "task": payload.task,
            "deliverables": payload.deliverables or [],
            "constraints": payload.constraints.dict() if payload.constraints else {},
            "assumptions": ["M2 stub – planner/synthesizer are minimal"],
        },
        "agent_roster": roster,
        "collab_log": [
            f"Selected mode = {mode} based on deliverables",
            "Loaded roles from data/roles.yaml",
        ],
        "deliverables": {
            "primary": primary_md
        },
        "risks": ["Outputs are draft-level; planning is heuristic-only"],
        "next_steps": [
            "Implement planner (M2+)",
            "Add OpenAI adapter for one step (M3)",
            "Add auth & Docker (M3)"
        ],
    }
```
