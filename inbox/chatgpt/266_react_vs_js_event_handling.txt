**Top task:** Explain differences between React event handling and standard DOM event handling in vanilla JavaScript.

---

**1. React vs. Vanilla JS event handlers (beginner-friendly)**  
- **Attachment:**  
  - *Vanilla JS*: use `element.addEventListener('click', handler)` or inline `onclick="handler()"`.  
  - *React*: use JSX props like `<button onClick={handler}>` (camelCase).  
- **Event object:**  
  - *Vanilla JS*: you get a native `MouseEvent`, `KeyboardEvent`, etc.  
  - *React*: you get a cross-browser `SyntheticEvent` that normalizes behavior.  
- **Preventing default:**  
  - *Vanilla JS*: returning `false` in some frameworks may work; otherwise call `event.preventDefault()`.  
  - *React*: always call `e.preventDefault()` inside your handler; returning `false` has no effect.  
- **Performance:**  
  - *Vanilla JS*: you might attach many listeners to individual nodes.  
  - *React*: uses event delegation at the root, so fewer native listeners are added automatically.
