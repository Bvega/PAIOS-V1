The Netlify build is failing due to a malformed `package.json`. Here’s how to fix it:

---

### ✅ Step-by-Step Fix:

1. **Open `package.json`** in your project root.

2. **Validate JSON Format**:
   - Use an online validator like: [https://jsonlint.com](https://jsonlint.com)  
   - Or validate locally via CLI:
     ```bash
     cat package.json | jq empty
     ```
     If you don’t have `jq`, install it or stick to online tools.

3. **Common Fixes to Look for**:
   - **Trailing commas**:
     ```json
     "scripts": {
       "start": "vite",
     } ← ❌ Invalid
     ```
   - **Unquoted keys** or values.
   - **Mismatched braces or brackets**.
   - **Line 32** is a clue from the Netlify error. Go directly there and check for syntax errors.

4. **Re-save file using UTF-8 (without BOM)** in your editor.

5. **Example of a Correct `package.json`** (truncated):
   ```json
   {
     "name": "recipe-discovery-app",
     "version": "1.0.0",
     "scripts": {
       "dev": "vite",
       "build": "vite build",
       "preview": "vite preview"
     },
     "dependencies": {
       "react": "^18.2.0",
       "react-dom": "^18.2.0"
     }
   }
   ```

6. **Commit the fix** and push:
   ```bash
   git add package.json
   git commit -m "Fix: corrected JSON syntax in package.json"
   git push origin main
   ```

7. **Re-trigger Netlify build** from the Netlify dashboard or wait for auto-build.
