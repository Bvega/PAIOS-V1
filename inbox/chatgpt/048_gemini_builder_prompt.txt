## 4) Data & API

### Models (Mongoose)
- **Tea**: `{ name, slug, description, origin, notes: string[], price: number, imageUrl, isPublished: boolean, createdAt }`
- **Order** (optional if Stripe): `{ items: [{ teaId, qty, unitPrice }], email, amount, stripeSessionId, status }`
- **User** (optional admin): `{ email, passwordHash, role }`

### REST Endpoints
- `GET /api/teas` → list published teas  
- `GET /api/teas/:slug` → single tea  
- `POST /api/cart/checkout` → if Stripe configured, create Checkout Session and return URL  
- `POST /api/contact` → send contact email (Resend), validate inputs  
- `POST /api/newsletter` → subscribe via Resend/Mailchimp  
- **Admin** (JWT-protected if enabled):  
  - `POST /api/auth/login` → returns token  
  - `POST /api/teas` | `PUT /api/teas/:id` | `DELETE /api/teas/:id`

> Include **Zod** or a simple validator middleware for request payloads.
