#### 🧠 Instructions Recap:

- If `discount` is provided (and valid), subtract it *before* applying tax:
  \[
  \text{totalCost} = ((\text{price} \times \text{quantity}) - \text{discount}) \times (1 + \text{taxRate})
  \]
- If `discount` is **not** provided, calculate total cost as usual.
- Input validation still applies: all inputs must be valid numbers, and discount must not make total negative.

---

### 🧪 Refactored Code:
```javascript
// Task 4: Refactoring for Reusability
function calculateTotalCost(price, quantity, taxRate, discount = 0) {
    if (
        typeof price !== 'number' || isNaN(price) ||
        typeof quantity !== 'number' || isNaN(quantity) ||
        typeof taxRate !== 'number' || isNaN(taxRate) ||
        typeof discount !== 'number' || isNaN(discount)
    ) {
        return "Invalid input.";
    }

    const subtotal = price * quantity;
    if (discount > subtotal) {
        return "Invalid input: discount exceeds subtotal.";
    }

    const totalCost = (subtotal - discount) * (1 + taxRate);
    return totalCost;
}

// Sample tests
console.log(calculateTotalCost(10, 2, 0.1));           // 22 (no discount)
console.log(calculateTotalCost(10, 2, 0.1, 5));         // 16.5
console.log(calculateTotalCost(10, 2, 0.1, 25));        // "Invalid input: discount exceeds subtotal."
```

### 📘 **What Does `1 + taxRate` Do in the Formula?**

The formula is:
```javascript
totalCost = (price * quantity - discount) * (1 + taxRate);
```

#### 🔍 Here's What It Means:
When you buy something, you often pay **sales tax** in addition to the product's base cost. The tax is **a percentage of the subtotal**, and we need to add that tax to the total.

Let’s say:
- Subtotal = $100
- Tax rate = 10% → written as `0.10` in decimal form

To get the **final total with tax**, you multiply the subtotal by **(1 + taxRate)**:
```javascript
total = 100 * (1 + 0.10) = 100 * 1.10 = 110
```

---

### 💡 Why Multiply by `1 + taxRate`?
- `1` = 100% of the original price (you’re always paying this)
- `taxRate` = the **extra percent** you're adding

So `1 + taxRate` gives you:
> **Original amount + tax**

---

### ✅ Real Example:
```javascript
price = 10, quantity = 2, taxRate = 0.1

subtotal = 10 * 2 = 20
total = 20 * (1 + 0.1) = 20 * 1.1 = 22
```

That’s why we use it — it's a shorthand for “add tax on top of the subtotal.”
