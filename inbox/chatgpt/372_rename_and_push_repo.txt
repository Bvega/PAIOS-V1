**Top Task**: Create and push to a newly named GitHub repo from your existing clone.

**Steps to rename and push your clone to a new repo:**

1. **Create the new repo on GitHub**  
   - Go to GitHub → “New repository” → enter your **new** repository name → click **Create**.

2. **In your local cloned repo, rename the existing `origin` remote**  
   ```bash
   git remote rename origin old-origin
   ```

3. **Add the new GitHub repo as `origin`**  
   ```bash
   git remote add origin https://github.com/<YOUR_USERNAME>/<NEW-REPO-NAME>.git
   ```

4. **Push your branches to the new remote with upstream tracking**  
   - If your main branch is called `main`:
     ```bash
     git push -u origin main
     ```
   - If it’s called `master`:
     ```bash
     git push -u origin master
     ```

5. **Verify on GitHub**  
   - Visit the new repo URL in your browser and confirm your code is there.

ASSISTANT:
**To list your branches in VS Code:**

1. **Using the Status Bar**  
   - Look at the bottom left corner of VS Code’s window.  
   - You’ll see your current branch name (e.g. `main`). Click on it to open the **branch picker**.  
   - A dropdown will show all **local** and **remote** branches.

2. **Using the Command Palette**  
   - Press `Ctrl+Shift+P` (Windows/Linux) or `Cmd+Shift+P` (macOS).  
   - Type and select **Git: Checkout to…** (it also lists branches). You’ll see all local branches and can switch or view them.

3. **Using the Integrated Terminal**  
   - Open the terminal (`Ctrl+` `` on Windows/Linux or ``Cmd+` `` on macOS).  
   - Run:
     ```bash
     git branch          # lists all local branches
     git branch -r       # lists all remote branches
     git branch --all    # lists both local and remote
     ```
