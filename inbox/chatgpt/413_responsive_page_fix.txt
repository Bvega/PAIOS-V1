ASSISTANT:
Here’s what each part does:

- `<!DOCTYPE html>`  
  Declares this document as HTML5, ensuring browsers render it in standards mode.

- `<html lang="en"> … </html>`  
  The root element. `lang="en"` tells browsers and assistive tech that the content is in English.

- `<head> … </head>`  
  Metadata and resources:
  - `<meta charset="UTF-8">`  
    Sets the character encoding so characters (including accented letters) display correctly.
  - `<meta name="viewport" content="width=device-width, initial-scale=1.0">`  
    Ensures the page scales properly on mobile devices by matching the viewport width to the device’s width.
  - `<link rel="stylesheet" href="styles.css">`  
    Pulls in your external CSS file for styling.
  - `<title>My Responsive Page</title>`  
    The text shown in the browser tab or window title.

- `<body> … </body>`  
  The visible content of the page:

  1. **`<header class="site-header"> … </header>`**  
     - Wraps the introductory content and main navigation.  
     - `<h1>My Responsive Page</h1>`: the main heading of your site.  
     - `<nav class="main-nav"> … </nav>`: a semantic container for your links.  
       - `<ul>` with `<li><a href="#sectionX">…</a></li>`: an unordered list of links that jump to sections within the page.

  2. **`<main class="content"> … </main>`**  
     - The primary content area of the page (only one `<main>` per page).  
     - Contains two `<section>` elements, each with:
       - `id="section1"` or `"section2"` so the nav links can target them.
       - `<h2>` sub‑headings for each section.
       - `<p>` paragraphs holding the actual content.

  3. **`<footer class="site-footer"> … </footer>`**  
     - Concludes the page with footer content (often copyright, links, etc.).
     - Here it simply has a `<p>` for your footer text.

---

**Why it’s “semantic” HTML:**  
- Elements like `<header>`, `<nav>`, `<main>`, `<section>`, and `<footer>` convey meaning about the structure instead of just using generic `<div>`s.  
- This improves accessibility (screen readers can navigate by sections), SEO (search engines understand your layout), and maintainability.
