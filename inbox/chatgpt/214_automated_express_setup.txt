how can we automate the basic setup for an express.js server?

--------------------------------------------------------------------------------

ASSISTANT:
Below is a simple, copy-and-paste one-file generator that bootstraps a clean Express.js project for you in seconds. Run it once and it will:

1. create a new folder,  
2. `npm init -y`,  
3. install Express, dotenv, and nodemon,  
4. scaffold a minimal but ready-to-grow folder structure,  
5. write sensible defaults into **server.js**, **.env**, **.gitignore**, and **package.json** scripts.

---

### 1 Save this file as `setup-express.js`

```js
#!/usr/bin/env node
/**
 * Quick Express project generator
 * Usage:  node setup-express.js my-app
 */
const { execSync } = require('child_process');
const { mkdirSync, writeFileSync, existsSync } = require('fs');
const path = require('path');

const appName = process.argv[2];
if (!appName) {
  console.error('❌  Provide a project folder name.\n   node setup-express.js my-api');
  process.exit(1);
}
if (existsSync(appName)) {
  console.error(`❌  Folder "${appName}" already exists. Choose another name or delete it.`);
  process.exit(1);
}

const run = (cmd, opts = {}) => execSync(cmd, { stdio: 'inherit', ...opts });

console.log(`📦  Creating ${appName}/ …`);
mkdirSync(appName);
process.chdir(appName);

run('npm init -y');

console.log('⏬  Installing dependencies …');
run('npm i express dotenv');
run('npm i -D nodemon');

console.log('📁  Scaffolding folders …');
['src', 'src/routes', 'src/controllers'].forEach(dir => mkdirSync(dir, { recursive: true }));

// ────────────────────────────────────────────────────────────────
// server boilerplate
const serverCode = `require('dotenv').config();
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

// Middleware
app.use(express.json());

// Example route
app.get('/', (req, res) => {
  res.json({ message: 'API is running 🚀' });
});

// Start server
app.listen(PORT, () => {
  console.log(\`Server listening on http://localhost:\${PORT}\`);
});
`;
writeFileSync('src/server.js', serverCode);

// .env
writeFileSync('.env', 'PORT=3000\n');

// .gitignore
writeFileSync('.gitignore', `node_modules
.env
`);

// patch package.json scripts
const pkgPath = path.join(process.cwd(), 'package.json');
const pkg = require(pkgPath);
pkg.scripts = {
  start: 'node src/server.js',
  dev: 'nodemon src/server.js'
};
writeFileSync(pkgPath, JSON.stringify(pkg, null, 2));

console.log(`✅  Done!

Next steps:
  cd ${appName}
  npm run dev     # runs with nodemon
  curl http://localhost:3000/
`);
```

---

### 2 Run the generator

```bash
node setup-express.js my-api
```

---

### 3 Fire it up

```bash
cd my-api
npm run dev
# → Server listening on http://localhost:3000
```

Feel free to extend the scaffold (add routers, controllers, tests, Dockerfile, CI, etc.)—the script gives you a clean, repeatable baseline you can invoke any time you need a fresh Express.js starter.

--------------------------------------------------------------------------------

ASSISTANT:
Yes. Here’s the clean-est way to make the generator available everywhere on Windows:

### 1  Save the script
* Put **setup-express.js** in `C:\Tools`

### 2  Add `C:\Tools` to your PATH  
1. Press **Win + R**, type `sysdm.cpl`, hit **Enter** → **Advanced** tab → **Environment Variables**.  
2. Under *User variables* select **Path** → **Edit** → **New** → paste `C:\Tools` → **OK** out of all dialogs.  
3. Close and reopen any terminal windows (or run `refreshenv` if you have Chocolatey).

### 3  (Optional but nicer) drop a one-liner wrapper  
Create a file called **express-init.cmd** in `C:\Tools` with only:

```bat
@echo off
node "%~dp0setup-express.js" %*
```

Because `*.cmd` files are executable, `express-init` will now resolve via PATH and forward all arguments to the Node script.

### 4  Use it from any folder
```powershell
# With the wrapper
express-init my-api

# Or, without the wrapper
node C:\Tools\setup-express.js my-api
```
Then:
```powershell
cd my-api
npm run dev
```
Server should start on **http://localhost:3000**.
