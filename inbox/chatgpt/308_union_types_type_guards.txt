### 📁 File: `dataProcessor.ts`

```ts
// Type alias for input array
type StringOrNumberArray = string[] | number[];

/**
 * Processes a string[] or number[] and returns a formatted string.
 * - For string[]: comma-separated uppercase strings
 * - For number[]: comma-separated numbers rounded to 2 decimals
 * 
 * @param data - Input array of strings or numbers
 * @returns A formatted string based on the input type
 */
function processData(data: StringOrNumberArray): string {
  if (isStringArray(data)) {
    return data.map(str => str.toUpperCase()).join(', ');
  } else if (isNumberArray(data)) {
    return data.map(num => num.toFixed(2)).join(', ');
  } else {
    throw new Error("Invalid data type");
  }
}

// Type guards
function isStringArray(data: StringOrNumberArray): data is string[] {
  return typeof data[0] === 'string';
}

function isNumberArray(data: StringOrNumberArray): data is number[] {
  return typeof data[0] === 'number';
}

// Example Usage
console.log(processData(["apple", "banana", "cherry"])); // "APPLE, BANANA, CHERRY"
console.log(processData([1.2345, 6.789, 10]));           // "1.23, 6.79, 10.00"
```

### 💡 Critical Thinking Reflection

#### ✅ **How union types and type guards help:**

1. **Safety through Explicitness:**
   - Using `string[] | number[]` restricts the function to only acceptable inputs.
   - Helps avoid silent bugs from unintentional or malformed data.

2. **Type Guards Enforce Logic Paths:**
   - `isStringArray()` and `isNumberArray()` confirm the input type at runtime.
   - This gives TypeScript the context it needs to narrow the type and avoid unsafe operations.

3. **Maintainability:**
   - If new types (e.g., `boolean[]`) are added in the future, extending logic is straightforward.
   - Keeping logic paths separate reduces complexity and cognitive load.

4. **Error Handling and Debugging:**
   - The `else` clause with an explicit `throw` makes handling unexpected input safer and easier to trace.

5. **Improves Developer Confidence:**
   - Future contributors clearly see how the function handles inputs and what outputs to expect.
