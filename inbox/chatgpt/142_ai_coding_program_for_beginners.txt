# AI for Coding — Beginner Program (10 lessons • 15 hours)  
Beginner-friendly, project-based, and 100% hands-on. The structure mirrors your outline (10 lessons, ~1.5h each), enhanced with approachable examples and clear deliverables. Stack: **JavaScript/TypeScript, Node.js, React (Vite), Express, Prisma + SQLite**, Netlify (FE) + Render/Railway (BE). Tools we’ll use along the way: **ChatGPT (incl. o3 & Canvas), Replit Agent (concepts & workflow), Ollama (local models)**.

---

## Program Overview
- **What you’ll master**
  - Generate complete functions/components from prompts
  - Turn cryptic errors into fixes with a repeatable debugging flow
  - “Secure by default” habits (validation, headers, secrets)
  - Ship a working prototype fast (scaffold → iterate → deploy)
  - Automate DB setup, API connections, and file structures
  - Prompting patterns for accurate, maintainable, and safe code
- **Capstone**: Ship a small **Full-Stack Notes/Tasks** app (React + Express + Prisma/SQLite) with auth-lite, deployed FE+BE.
- **Certificate**: Awarded after passing rubric (below) and submitting a deployed capstone + repo.

---

## Lesson-by-Lesson Plan

### 1) Setup & Mindset: “Your AI Pair-Programmer” (1.5h)
**Goals**: Install **VS Code**, **Node.js**, **Git**; learn the “Think → Prompt → Run → Review” loop.  
**Do**  
- Install & test: `node -v`, `npm -v`, `git --version`.  
- Create folder, init repo: `git init`, minimal **README**.  
- First prompt: “Generate a JS function `greet(name)` and a tiny test.”  
- Run and tweak with ChatGPT until the test passes.  
**Deliverable**: `greet.js`, `greet.test.js`, README with “What I changed and why.”

---

### 2) Prompting for Code: Components & Functions (1.5h)
**Goals**: Generate **React** components + small utility functions with tight specs.  
**Do**  
- Vite React app: `npm create vite@latest` → React + TS → `npm i && npm run dev`.  
- Build a **Counter** component (increment, decrement, reset) solely by prompting.  
- Add a utility: `formatCurrency(amount, locale)`.  
- **Pattern**: Role + Objective + Constraints + Examples + Tests.  
**Deliverable**: Counter with tests (Vitest) + `formatCurrency` with edge cases.

---

### 3) AI Debugging Clinic: From Error → Fix (1.5h)
**Goals**: Turn vague errors into actions.  
**Do**  
- Intentionally break code: `Cannot read properties of undefined`, bad imports, missing deps.  
- Triage checklist: **Reproduce → Read stack → Narrow → Minimal repro → Ask AI**.  
- Prompt pack: “Here’s the stack trace + minimal repro; explain root cause and give a tested fix.”  
**Deliverable**: Debug log (what failed, fix, proof) + a reusable **debugging checklist**.

---

### 4) Secure by Default (1.5h)
**Goals**: Bake in validation and harden Express from day one.  
**Do**  
- Scaffold Express API (`/api/notes`). Add **Helmet**, **CORS**, **rate limit**.  
- Request validation with **Zod** (or Yup).  
- Secrets via `.env`, never commit; show why.  
- Compare insecure vs secure routes and measure behavior.  
**Deliverable**: `server.js` with secure middleware + Zod schemas + `.env.example`.

---

### 5) Prototype in 60 Minutes (1.5h)
**Goals**: Ship fast with a walking skeleton.  
**Do**  
- From blank → **Notes prototype**: FE form + list, BE in-memory store.  
- Prompt ChatGPT: “Make the smallest end-to-end slice possible; no extras.”  
- Add a basic test for one route.  
**Deliverable**: Running prototype + 1 test + “Next slice” notes.

---

### 6) Automate DB, API & File Structures (1.5h)
**Goals**: Let AI scaffold the boring parts.  
**Do**  
- Add **Prisma + SQLite**; `npx prisma init`, model `Note`, migrate, seed.  
- Generate CRUD routes + service layer; prompt AI for clean file structure.  
- `.env` template; scripts: `dev`, `db:reset`.  
**Deliverable**: DB-backed CRUD API + documented project structure.

---

### 7) Agents 101: Replit Agent Workflow (1.5h)
**Goals**: Understand **plan → execute → review** loops and safe boundaries.  
**Do**  
- Write an **Agent Brief**: objective, constraints, files it may touch, “stop if…” rules.  
- Small task: “Create a service function and its unit test.” Review diffs.  
- Discuss rollback, PR review, and human-in-the-loop.  
**Deliverable**: Agent brief + a successfully generated service + test.

---

### 8) Advanced ChatGPT: o3 & Canvas (1.5h)
**Goals**: Use **o3** for reasoning; use **Canvas** for UI-first iteration.  
**Do**  
- UI sketch in Canvas → component tree → paste directly into project.  
- System prompt templates: coding style, security rules, test-first.  
- “Self-critique” loop: ask o3 to add tests and find edge cases.  
**Deliverable**: Canvas snapshot (export) + merged component updates + extra tests.

---

### 9) Local Models with Ollama (1.5h)
**Goals**: Offline/private workflows.  
**Do**  
- Install **Ollama**; run a lightweight code model.  
- Compare outputs: ChatGPT vs local model; when to prefer each.  
- Create a **privacy-first** prompt pack (no secrets; summarize/obfuscate).  
**Deliverable**: Local run log + comparison notes + safe-prompt template.

---

### 10) Capstone & Deploy (1.5h)
**Goals**: Finish, harden, and deploy.  
**Do**  
- Feature set: notes/tasks, tags, search, simple auth-lite (session cookie or token).  
- Tests: at least 3 key endpoints + 1 React component.  
- Deploy: **Netlify** (FE) + **Render/Railway** (BE); env vars; post-deploy smoke tests.  
- Add a “Setup script” in README (copy/paste steps A→Z).  
**Deliverables**: Public URLs, repos, README with setup, and a short demo video (2–3 min).

---

## Prompt Packs (copy, adapt, reuse)

**Spec → Code**
> “You are my senior JS/TS engineer. Generate a React `<Counter/>` with props `initial`, `step`. Include TypeScript types, accessibility (button labels), and Vitest tests that cover increment, decrement, and reset. Keep functions under 20 lines and add 3 doc comments.”

**Error Triage**
> “Here’s a minimal repro and stack trace. 1) Explain root cause, 2) propose 2 fixes with tradeoffs, 3) give the exact diff for the chosen fix, 4) a test to prevent regressions.”

**Secure API**
> “Harden an Express route `/api/notes` with Helmet, CORS (FE domain only), and Zod validation for `{title: string<=80, body?: string}`. Provide code + tests + a `.env.example`.”

**Scaffold DB**
> “Create Prisma schema for `Note { id, title, body, tags[], createdAt }`. Add seed data, CRUD services, routes, and scripts `db:migrate`, `db:seed`, `db:reset`. Show final file tree.”

**Agent Brief**
> “Objective: Add a `searchNotes(query)` service and unit tests. Constraints: Only edit `/server/src/services/*` and `/tests/*`. Stop if tests fail; request approval.”

## Assessment Rubric (pass = certificate)
- **Functionality (30%)**: Core features work; happy path + edge cases.
- **Code Quality (20%)**: Clear structure, TS types, small functions, comments.
- **Security & Env (15%)**: Validation, headers, secrets handled, no hard-coded creds.
- **Testing (15%)**: Unit tests for at least 3 key paths; one FE test.
- **Prompting Mastery (10%)**: Reusable prompts; self-critique loops.
- **Docs & Deploy (10%)**: Clean README, env setup, working live URLs.
